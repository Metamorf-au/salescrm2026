Implementation Plan — Week by Week

Week 1 — Foundation & Authentication

25 hrs

Set up the entire backend infrastructure. By end of week, all users can log in and the database is ready for data.

  • Set up Supabase project — database, auth, storage, confirm Sydney region
  • Design database schema — contacts, deals, notes, calls, activity_log, users tables with correct relationships and indexes
  • Create all tables — Run SQL migrations to build the schema
  • Configure authentication — Email/password login, session management
  • Set up Row Level Security policies — Reps see own data, managers see their team, admin sees everything
  • Create 10 user accounts — With correct roles assigned
  • Connect frontend to Supabase — Install client library, configure environment variables, build login/logout flow
  • Verify auth works — Test login as each role type (rep, manager, admin) and confirm RLS policies enforce correct data access

Deliverable: All 10 users can log in. Database schema complete. RLS policies verified.

Week 2 — Wire Frontend to Real Data + DataTools

25 hrs

Replace all mock/hardcoded data with live Supabase queries. Integrate DataTools for address autocomplete and validation. By end of week, the app is fully functional with real data.

Contacts

  • Contact list & search — Replace mock CONTACTS array with Supabase query, pagination for 6,000+ records
  • Create contact — Form writes to database with DataTools address autocomplete (rep types, suggestions appear, structured address saved)
  • Edit contact — Update existing records
  • Call logging — Log calls against contacts, store outcome and summary
  • Notes with reminders — Create/read typed notes (general, follow-up, meeting, pricing, internal) with reminder dates

Pipeline & Deals

  • Deal list by stage — Kanban board reads from database, filtered by rep/stage
  • Create deal — New deal form writes to database, linked to contact
  • Stage transitions — Move deals between stages (discovery → quote request → quote sent → won/lost)
  • Won/Lost flows — Won celebration modal + stage update, Lost with reason capture
  • Quote timestamps — Auto-set quoteRequestedAt and quoteSentAt on stage transitions

Activity & KPIs

  • Activity log — Auto-writes on every action (call logged, deal created, deal won, etc.)
  • Rep KPI dashboard — 6 metrics calculated from real data
  • Manager KPI dashboard — 10 aggregated metrics from real data, quote turnaround from timestamps, weighted pipeline value
  • Weekly scorecard — Per-rep metrics table with real data

Deliverable: Fully functional app with real data. DataTools address autocomplete working on contact forms.

Week 3 — Import, Polish & Deploy

25 hrs

Import the 6,000 contacts, validate addresses via DataTools, deploy to production, and onboard users.

CSV Import

  • Build CSV import tool — Supabase Edge Function to parse, validate, and bulk insert contacts
  • Column mapping — Map spreadsheet columns to database fields
  • DataTools address validation — Run imported addresses through DataTools to standardise and correct (e.g. "Melb" → "Melbourne, VIC 3000")
  • Duplicate detection — Flag or merge duplicate contacts during import
  • Import the 6,000 contacts — Run the actual import with validation report

Testing & Deployment

  • End-to-end testing — Test all flows as rep, manager, and admin roles
  • Mobile testing — Verify responsive layout works on phones/tablets
  • Deploy to Vercel — Production environment, environment variables, domain config
  • Onboard 10 users — Send login credentials, brief walkthrough
  • Bug fixes from initial use — Address any issues found during first days of use

Deliverable: Production system live. 6,000 contacts imported and validated. 10 users onboarded.

Week 4 — Buffer & Stabilisation

If needed

Reserved for issues discovered during real-world use. May not be needed if Weeks 1–3 go smoothly.

  • Edge cases — Issues discovered from real daily use by the team
  • Data cleanup — Fix any import issues, address validation gaps, duplicate contacts
  • Feedback adjustments — Minor UI tweaks based on user feedback
  • Performance tuning — Optimise slow queries if any (6,000 contact search, dashboard aggregations)

DataTools Integration — Detail

DataTools address validation and autocomplete is integrated in two key areas to improve data quality and rep efficiency.

FeatureWhereHow It Works
Address Autocomplete New Contact form, Edit Contact form Rep starts typing an address, DataTools returns suggestions. Rep selects one, full structured address (street, suburb, state, postcode) is auto-populated. Faster entry, consistent formatting, fewer errors.
Bulk Address Validation CSV import (6,000 contacts) During import, each address is validated via DataTools API. Invalid or incomplete addresses are flagged in a report. Valid addresses are standardised to a consistent format before saving.
Ongoing Validation Phase 2 (optional) Periodic batch validation to catch addresses that have degraded. Not in scope for Phase 1 but the foundation will support it.

Required from you: DataTools API key and confirmation of which products are on your plan (address autocomplete, address validation, or both). This is needed before Week 2.

Security Overview

For a 10-user internal sales tool, the following security measures are in place from day one.

LayerDetail
Encryption at restAll database data encrypted on disk (AES-256)
Encryption in transitAll connections via TLS/SSL (HTTPS)
AuthenticationSupabase Auth — password hashing (bcrypt), secure session tokens
Row Level SecurityDatabase-level policies enforce who can see/edit what — reps own data, managers team data, admin all data
Data residencyHosted on AWS Sydney (ap-southeast-2) — data stays in Australia
InfrastructureSOC 2 Type II compliant (Supabase)
Daily backupsAutomated on Supabase Pro plan

Phase 2 security upgrades (when needed):

  • Multi-factor authentication (MFA) — Supabase supports natively, configuration only
  • Audit logging — who accessed/changed what and when
  • IP allowlisting — restrict access to office/VPN networks
